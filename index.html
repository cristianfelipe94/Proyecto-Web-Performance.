Web Performance

¿Qué es?

Es la disciplina que se encarga de analizar un sitio web para recomendar cambios que ayuden a optimizar la velocidad de carga de la misma. La velocidad es un elemento crítico para ejecutar un sitio web exitoso y siempre debe ser una prioridad para los administradores de sitios web. Los sitios web de carga más rápida se benefician de un mejor compromiso del usuario. La velocidad de carga de una Web es un factor a tener en cuenta para el posicionamiento en buscadores.

Compresión de archivos.

A medida que las páginas web crecen en complejidad, también lo hacen sus archivos y posteriormente, sus tiempos de carga. Por eso hay que tener en cuenta optimizar imagenes, videos, audio y comprimir archivos PDF de ser necesario, para tener una buena experiencia en el sitio web.

Herramientas recomendadas a utilizar:

Compressor.io: Es un optimizador de imagenes basado en servicios de la nube.

Convert.io: Es una herramienta gratis que nos permite comprimir archivos PDF en tres calidades distintas.

Online Audio Converter: Es un sencillo online, puede convertir archivos de formatos pesados a otros no tanto, como .wav en .mp3, se puede reducir la calidad de un mp3 con pérdida en resolución casi imperceptible.

Handbrake: Es un programa gratuito que permite cambiar el formato de los videos sin limites de tamaño, ya que otros servicios o softwares nos da un límite de tamaño al subir el archivo.

Optimización de almacenamiento en caché

Aprovechar el caché del navegador es crucial. En el caso de los Websites, el caché puede salvar una copia de imágenes, hojas de estilo, JavaScript. La próxima vez que el usuario necesite ese recurso el browser no tienen que descargarlo de nuevo, esto ahorra tiempo al servidor y hace que todo sea más rápido.

Reducción del código.

Hoy en día la web moderna requiere y exige de archivos de poco tamaño, para que los dispositivos móviles puedan leer las páginas a mayor velocidad, consumir menor ancho de banda y disminuir en lo posible el consumo de datos. Existen herramientas y buenas prácticas que nos pueden ayudar a reducir el código.

Optimización de la imagen.

Como se nombra anteriormente, optimizar y comprimir los archivos es un impacto importante en la carga de las paginas web, por eso no cargue las fotos originales en su sitio web, son demasiado pesadas ya que contienen mucha información adicional (como metadatos acerca de la cámara, fecha, ubicación...) que pueden ser eliminados sin que eso suponga una pérdida de calidad. Use herramientas que reduzcan el tamaño de la imagen mientras que la calidad permanece más o menos igual.

Herramienta recomendada a utilizar:

Riot (Radical Image Optimization Tool): Es una herramienta para optimizar las imágenes desde tu ordenador antes de subirlas. Es gratis para Windows, pero si usa MAC, se puede usarImageOptim.

Redirecciones

Los redireccionamientos son enemigos del rendimiento, evítalos siempre que sea posible. Las redirecciones son, de forma resumida, enviar a los visitantes de una web a otra diferente o de una zona de una web a otra de manera automática.

Monitoreo de sitios web en la nube

Hay muchas ventajas de monitorear tu página en un host basado en la nube.

  • Supervisar la disponibilidad
  • Vigilar los tiempos de carga
  • Monitorizar los procesos
  • Sistema de notificaciones y alertas

Un sistema de monitoreo de sitios Web apunta cada rincón. Por lo tanto, ahorrará un tiempo considerable para llegar a la raíz de los problemas y podrá corregir los errores más rápidamente.

Certificado HTTPS

Los HTTP Son protocolos o lenguajes para pasar información entre servidores web y clientes. Con una conexión HTTP estándar, es posible que terceras partes no autorizadas puedan acceder a esta información. El HTTPS es solo el primer paso de una larga lista de cambios que vendrán pronto para proteger mejor la privacidad de los usuarios.

Rendimiento de fuentes web

Las desventajas de las fuentes web, como Google Fonts, son que agregan solicitudes HTTP adicionales a recursos externos. Basado en el browser, trate de usar fuentes nativas y mantenga un uso mínimo o nulo de fuentes con caracteres especiales.

Protección Hotlink

Hotlinking es cuando un propietario de un sitio web enlaza directamente su sitio con imágenes o archivos multimedia (pertenecientes a tú sitio) y la incluye en su página web. La protección Hotlink te ahorrará ancho de banda al prohibir que otros sitios muestren tus imágenes.

Errores 404.

Cualquier archivo faltante genera un error de HTTP 404. El valor 404 advierte de que la URL solicitada no se encuentra en el servidor, como cuando intentas acceder a una página que ha sido borrada o escribes mal la dirección en el navegador.

Los errores 404, pueden ser bastante agotadores para el servidor. Existen varias aplicaciones recomendadas para detectar este tipo de errores, cuya función principal es localizar los enlaces que estan fallando en tu página o enlaces rotos.

Colocar los CSS arriba y los Scripts abajo

Debemos colocar los llamados del CSS dentro del porque si los colocamos abajo, la página se empezará a renderizar sin estilos hasta que se descargue el archivo. Por el contrario, los archivos JavaScript deberían ser agregados al final porque si no bloquean el renderizado mientras se descargan y se ejecutan. Estos deben colocarse antes de la etiqueta de cierre </body>.

Identacion, tabulaciones y comentarios

Tanto la tabulacion, identacion y los comentarios son una buena práctica para mantener el código legible ya que el navegador puede prescindir de ellos.

Elimina cookies innecesarias

Las cookies son pequeños ficheros de datos que se guardan en el navegador del usuario. Contienen información necesaria para facilitar el proceso de navegación a través de las Web. Cada vez que se abre una nueva página durante la navegación se tiene que enviar la Cookie, procesarla, posiblemente modificarla y volverla a guardar en el ordenador del usuario.
